# Landsat Radiation Calculator (L8/L9)

Автоматический расчёт параметров атмосферной коррекции (Tau, Lu, Ld) для тепловых каналов спутников Landsat 8 и 9 с использованием метода Radiative Transfer Equation (RTE).

## Описание
Проект предназначен для пакетной обработки сцен Landsat. Скрипт автоматически извлекает метаданные из файлов `_MTL.txt`, определяет необходимые атмосферные условия (водяной пар) на основе данных реанализа ERA5 или Open-Meteo API и рассчитывает параметры, необходимые для восстановления температуры подстилающей поверхности (LST).

## Ключевые особенности
*   **Умный механизм Fallback**: Приоритетная загрузка точных профилей ERA5 из локальных файлов (NetCDF). При их отсутствии — автоматический запрос к Open-Meteo API. Если API недоступен — использование дефолтных значений.
*   **Надёжность**: Встроенный механизм повторных попыток (Retry) при сбоях сети или таймаутах API (до 5 попыток).
*   **Пакетная обработка**: Поддержка обработки директорий с множеством сцен и автоматическое создание сводной CSV-таблицы.
*   **Гибкая конфигурация**: Все константы (координаты региона, уровни давления, LUT-таблицы) вынесены в `config/settings.py`.
*   **Детальные отчёты**: Генерация подробных TXT-отчётов для каждой сцены с указанием источника данных.

  ```
  key: <ваш_uid>:<ваш_api_key>
  ```

## Использование

### Базовый запуск
Обработка всех сцен в папке `data` (с попыткой использования ERA5):
```bash
python -m src.main ./data
```

### Отключение ERA5 (только API)

Если вы хотите использовать только Open-Meteo (без загрузки локальных файлов ERA5):
```bash
python -m src.main ./data --no-era5
```

### Ручной ввод водяного пара

Если вы хотите задать значение водяного пара (Total Column Water Vapor) вручную для всех сцен:
```bash
python -m src.main ./data --vapor 2.15
```

## Логика работы

Процесс обработки одной сцены выглядит следующим образом:
1. **Подготовка данных и пакетный запуск**  
    В папку `./data` необходимо поместить файлы метаданных (`*_MTL.txt`) интересующих вас снимков Landsat 8/9. Скрипт работает в полностью автоматическом пакетном режиме: при запуске он рекурсивно сканирует указанную директорию, находит все доступные MTL-файлы и формирует очередь на обработку, не требуя ручного перечисления путей или интерактивного ввода.
2. **Парсинг метаданных сцены**  
    Для каждой сцены скрипт автоматически извлекает из MTL-файла ключевые параметры: географические координаты центра сцены (среднее между углами), точную дату и время съёмки (UTC), углы солнечной геометрии (зенит, азимут), облачность и калибровочные константы теплового канала (K1, K2). На основе даты автоматически определяется необходимый год для загрузки атмосферных профилей.
3. **Получение атмосферных данных (каскадный fallback)**  
    Процесс определения содержания водяного пара (TCWV) работает по строгому приоритетному алгоритму:
    - **Шаг A: ERA5 (CDS API).** Скрипт проверяет наличие локальных файлов профилей (`era5_profiles_YYYY.nc`). Если файлы отсутствуют, выполняется попытка загрузки через Climate Data Store API. При успехе TCWV рассчитывается путём вертикальной интеграции удельной влажности по всем уровням давления.
    - **Шаг B: Open-Meteo Historical API.** Если доступ к CDS невозможен (например, не настроен `.cdsapirc`), система переключается на открытый архив Open-Meteo. Запрашиваются приземная температура и относительная влажность для координат.
    - **Механизм повторных попыток.** В случае сетевого сбоя, таймаута или пустого ответа от API, скрипт автоматически повторяет запрос до 5 раз с фиксированной задержкой 2 секунды между попытками. Это гарантирует устойчивость к кратковременным перебоям связи.
    - **Аварийный fallback.** Если после 5 попыток данные получить не удалось, автоматически подставляется научно обоснованное значение по умолчанию: **`1.8 г/см²`** (климатическая норма содержания водяного пара для субарктического лета в районе Норильска). В терминале и отчёте такой случай помечается как `Default`.
4. **Расчёт атмосферных параметров**  
    После определения водяного пара скрипт рассчитывает три ключевых параметра атмосферной коррекции: пропускание атмосферы (τ), уходящее излучение (Lu) и приходящее излучение (Ld).
    
    **Методология:** Расчёт выполняется методом интерполяции предварительно рассчитанных таблиц MODTRAN/6S (Look-Up Tables, LUT) для теплового диапазона 10.6–11.19 мкм. Скрипт автоматически выбирает соответствующий атмосферный профиль (например, `SubarcticSummer` для широт >60° в период июнь–сентябрь) и выполняет линейную интерполяцию значений по полученному водяному пару.  
    
    **Формула:** Полученные параметры используются для изоляции атмосферной составляющей в уравнении радиационного переноса (RTE), лежащего в основе восстановления температуры подстилающей поверхности (LST): 

$$
\boxed{
L_\lambda = \tau \cdot \varepsilon \cdot B(T_s) + L_u + \tau \cdot (1 - \varepsilon) \cdot L_d
}
$$

где:
- $L_\lambda$ — спектральная яркость на датчике, Вт·м⁻²·ср⁻¹·мкм⁻¹
- $\tau$ — пропускание атмосферы
- $\varepsilon$ — излучательная способность поверхности
- $B(T_s)$ — функция Планка при температуре поверхности $T_s$
- $L_u$ — Downwelling Radiance
- $L_d$ — Upwelling Radiance

Скрипт предоставляет готовые $\tau$, $L_u$, $L_d$, которые можно использовать далее для расчёта Ts (LST).

5. **Сохранение результатов**  
	По завершении расчётов для каждой сцены в папке `./output` генерируется детальный текстовый отчёт (формат имени: `atm_params_YYYY-MM-DD_Источник.txt`), содержащий все входные параметры, источник погодных данных, границы интерполяции LUT и итоговые значения. Дополнительно формируется сводная таблица `batch_results.csv` с разделителем `;`, готовая для прямого импорта в Excel или статистической обработки.
